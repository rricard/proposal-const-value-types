ECMAScript Data Types and Values

ECMAScript Language Types

The Record Type

The Record type is the set of all sorted sequences of zero or more pairs of key and data up to a maximum length of 253 - 1 elements. Pairs have the form { [[Key]], [[Value]] }, where [[Key]] is a String and [[Value]] is any primitive (including Record and Tuple). Pairs are sorted by [[Key]] in code unit order. Operations which create Record values are responsible for setting this up appropriately. The List of pairs is never modified.

The Record type is used to represent the association of multiple primitive values together. Each pair is regarded as occupying a position within the sequence. These positions are indexed with nonnegative integers. The first pair (if any) is at index 0, the next element (if any) at index 1, and so on. The length of a Record is the number of pairs within it. The empty Record has length zero and therefore contains no elements.

RecordTypeCreate ( _keysList_, _valuesList_ )

This abstract operation RecordTypeCreate takes a _keysList_ and a _valuesList_ argument. It is used to specify the creation of new Record value types. It performs the following steps when called:

1. Assert: Type(_keysList_) is a List of Strings. 1. Assert: Type(_valuesList_) is a List of primitive values. 1. Assert: _valuesList_.length == _keysList_.length 1. Let _L_ be an empty List. 1. For each _i_ from 0 to _keysList_.length: 1. Add { [[Key]]: _keysList_[_i_], [[Value]]: _valuesList_[_i_] } to _L_ 1. Sort _L_ by [[Key]] using code unit order 1. Return _L_

The Tuple Type

The Tuple type is the set of all ordered sequences of zero or more primitive type values up to a maximum length of 253 - 1 elements. Those primitive type values can include record and tuple values. The Tuple type is used to represent a series of multiple primitive values together. Each element is regarded as occupying a position within the sequence. These positions are indexed with nonnegative integers. The first element (if any) is at index 0, the next element (if any) at index 1, and so on. The length of a Tuple is the number of elements (i.e., primitive values) within it. The empty Tuple has length zero and therefore contains no elements.
